## The pre-commit gate (read before committing C++/Swift here)

Lives at `.githooks/pre-commit`, activated via `git config core.hooksPath
.githooks` (must be re-run on a fresh clone). It is version-controlled.

It does, on staged files:
1. **Format** (clang-format for C/C++/ObjC++, swiftformat for Swift) and re-stage.
2. **swiftlint** on staged Swift — blocks on **error**-severity only. Many
   non-blocking *warnings* remain (file length, `eq` enum name, `master` inclusive-
   language, param counts) — these are expected and do NOT fail the gate.
3. **clang-tidy** on staged C++/ObjC++/headers — blocks on any error.

Gotchas the next session WILL hit:
- **clang-format runs BEFORE clang-tidy**, and it reformats `.h` files (it skips
  `.mm` as ObjC). If you suppress a clang-tidy finding with a trailing `// NOLINT`
  on a multi-line declaration in a header, clang-format can re-wrap the line and
  move the comment off the diagnostic's anchor line → suppression silently fails.
  Use `// NOLINTBEGIN(...) / // NOLINTEND(...)` blocks in headers (see the Core
  Audio listener decl in `CoreAudioDevice.h`).
- The hook is **bash 3.2**-compatible (macOS system bash). Empty-array expansion
  under `set -u` throws — avoid `"${arr[@]}"` on possibly-empty arrays.
- clang-tidy must run as `-x objective-c++ -std=gnu++2b -fobjc-arc -fblocks
  -I Sources/AudioDSP/include -isysroot "$(xcrun --show-sdk-path)"`. The
  `gnu++2b` + ARC + blocks flags matter (Apple `__typeof`/blocks won't parse
  otherwise). There are **no clang-tidy exemptions left** — the whole AudioDSP
  tree is gated.

## `.clang-tidy` / config decisions

- C++ standard: **gnu++2b** (`Package.swift` `cxxLanguageStandard: .gnucxx2b`).
  Verified available on this toolchain: `std::span`, `std::mdspan`, `std::expected`,
  ranges, `<format>`. NOT in Apple libc++ yet: `std::float32_t` (`<stdfloat>`),
  `std::generator` (`<generator>`) — don't design around them.
- Disabled clang-tidy checks (incompatible with `-fno-exceptions` RT audio or too
  noisy): the three `pro-bounds-*`, `bugprone-easily-swappable-parameters`,
  `bugprone-exception-escape`. Config: `VariableCase=camelBack`, k-prefixed
  constexpr accepted, cognitive-complexity threshold 25, RBJ single-letter DSP
  names (A/Q/w0) allowed, structural float values 0/1/2 ignored.
